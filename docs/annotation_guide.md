# Annotation guide

Разметка нужна для обучения и проверки качества moderation pipeline. Финальная ручная разметка имеет приоритет над prediction модели и действием бота.

## Основные поля

Каждый пример должен иметь:

- `labels` — один или несколько классов;
- `primary_label` — главный класс;
- `severity` — уровень от `0` до `5`;
- `recommended_action` — рекомендуемое действие;
- `source` — происхождение примера;
- `annotator_confidence` — уверенность разметчика;
- `needs_context` — нужен ли контекст соседних сообщений.

## Multi-label

Разметка должна поддерживать несколько labels на одно сообщение. Например, рекламное сообщение может одновременно иметь признаки спама, приглашения и скама.

## Severity

- `0` — безопасно.
- `1` — слабый сигнал.
- `2` — низкий риск.
- `3` — средний риск.
- `4` — высокий риск.
- `5` — критичный риск.

## Действия

Action описывает реакцию системы, а не тип нарушения. Label и action нельзя смешивать.

Базовые действия: ignore, log, review, warn, delete, delete with warn, timeout, ban.

## Источники данных

Рекомендуемые значения `source`: `real_safe`, `real_moderated`, `test_channel`, `manual_synthetic`, `public_dataset`, `ai_generated`.

## Правила качества

1. Не обучать модель только на действиях бота.
2. Moderator feedback имеет приоритет над prediction.
3. Если нужен контекст, ставить `needs_context=true`.
4. Если разметчик сомневается, пример идёт в review bucket.
5. Перед export приватные данные должны быть замаскированы.
6. В training export не должны попадать примеры без source, labels и severity.
